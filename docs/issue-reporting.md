# Issue Reporting

Foundry uses public issue intake in the repository below:

- https://github.com/FoozleCC/Foundry-Construct-Collection/issues/new/choose

## Use the right form

- Bug report: for reproducible problems in released addon behavior.
- Feature request: for improvements or API requests.

## Bug report quality checklist

Include all of the following:

1. Addon name and addon version.
2. Construct version.
3. Exact reproduction steps.
4. Expected result and actual result.
5. Minimal reproducible project file or download link.
6. Relevant logs, screenshots, or video.

## Repro project guidance

A good repro project should:

1. Contain only the minimum objects/events needed to reproduce the issue.
2. Use clearly named layout and event sheet entries.
3. Avoid unrelated third-party addons when possible.
4. Reproduce on first preview without additional setup.

Issues without a reproducible project may be labeled `needs-repro` until enough information is provided.
